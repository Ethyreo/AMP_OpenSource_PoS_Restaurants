# Docs

This folder is the long-term project memory for `AMP PoS`. It is intended to help future sessions recover product intent, architectural choices, operational history, and the current app state without relying on chat history.

## Start Here

- `product/2026-03-23-app-plan.md`
  The original product plan and requirement direction that shaped the build.
- `journey/README.md`
  End-to-end project journey from first scaffold to the current local-first Android-ready app.
- `architecture/README.md`
  Current runtime architecture, storage model, Android shell details, and key source files.
- `operations/README.md`
  The working checklist and operator-facing implementation notes.
- `operations/SESSION-LOG.md`
  High-signal build log of the major bugs, regressions, fixes, and stabilization work completed so far.
- `features.md`
  Current feature inventory in product terms.

## Folder Purpose

- `product/`
  Requirements, design direction, feature clarifications, and future product updates.
- `journey/`
  Chronological implementation narrative and milestone context.
- `architecture/`
  Runtime choice, storage notes, Android wrapper details, backup model, and technical constraints.
- `decisions/`
  Important tradeoffs and design decisions worth preserving.
- `meetings/`
  Stakeholder notes, action items, and future review records.
- `operations/`
  Deployment notes, bug logs, support context, device observations, and rollout checklist history.
- `hardware/`
  Android device, printer, and store-floor hardware notes.
- `screenshots/`
  Visual snapshots for release notes, design references, and regression comparison.

## Current Snapshot

- Product name: `AMP PoS`
- Delivery mode: local-first web app packaged in an Android WebView shell
- Primary repo branch: `main`
